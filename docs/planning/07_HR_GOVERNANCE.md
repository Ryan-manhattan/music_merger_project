## 2) 결재(승인) 게이트 규칙
다음 항목은 **사장님 결재 없이는 절대 진행 금지**(스펙/기획 변경 포함).
- 중심 컨텐츠 범위 변경(현재 결재: **YouTube + SoundCloud**)
- IA/핵심 플로우 변경(목록/작성/상세의 의미 변경)
- 데이터 정책/필드/검증 규칙의 의미 변경
- “테마 비정합 기능”의 삭제/대수술(현재는 보존 + 우선순위 제외)
- 새 플랫폼/대형 의존성/외부 서비스 추가

결재 요청 작성 규칙(필수):
- 옵션은 **숫자**로 제시
- 옵션 중 **추천 1개**를 명시
- 리스크/비용/되돌리기 가능성 포함
- 보고서는 `docs/planning/00_REPORT_TEMPLATE.md` 포맷을 따른다

## 3) 커뮤니케이션/진행 방식(에이전트 공통)
- **모든 답변은 한글**
- 에이전트가 작업을 “진짜로 수행(수정/실행/추가)”하기 전에는 반드시 승인 상태가 명확해야 한다.
- 진행 여부 질문 시: **번호 옵션 + 추천**을 짧게 제시

(사장님 지시어 예시)
- `ㄱ` = 진행
- `ㅂㄱ` = 보고만(작업 수행 금지)
- `ㄱㄱㄱ` = 질문 없이 전체 자동 진행

## 4) 작업 사이클(표준 프로세스)
모든 작업은 아래 순서를 따른다.

### 4-1) 착수 전(필수)
- **작업 범위**(무엇을/왜/어디를) 1~3줄로 정리
- 결재가 필요한 변경이 있는지 체크(있으면 결재 보고 먼저)
- 리스크(데이터 손상/보안/배포 영향) 1~3줄

### 4-2) 구현/수행 중(필수)
- 변경은 “최소 단위”로 쪼개서 진행
- 오류 발생 시:
  - 원인 분석을 먼저 기록
  - 해결 **옵션 1/2/3 + 추천** 제시
  - 승인 없이 즉시 조치하지 않는다

### 4-3) 완료 후(필수)
- 변경 요약(핵심 3줄)
- 테스트/검증 결과(무엇을 어떻게 확인했는지)
- 영향 범위(페이지/엔드포인트/DB/문서)
- 필요한 경우 문서 동기화(추가/append)

---

## 5) 산출물/기록 규칙
### 5-1) 최종 보고(필수)
- 저장 위치: `docs/planning/reports/`
- 파일명: `YYYY-MM-DD_HH-MM-SS__REPORT.md`
- 포맷: `docs/planning/00_REPORT_TEMPLATE.md`
- 원칙: **누적 저장(삭제/덮어쓰기 금지)**

### 5-2) 회의록(필요 시)
- 저장 위치: `docs/planning/meetings/`
- 파일명: `YYYY-MM-DD_HH-MM-SS__MEETING.md`

### 5-3) 작업 로그(권장)
- 기획 작업 로그: `docs/planning/05_WORK_LOG.md`에 append

---

## 6) 문서(.md) 관리 규칙
- 기존 내용 **삭제 금지**: 수정이 필요하면 “추가 섹션/추가 문장”으로 남긴다(append)
- 문서는 최소 용량/최소 개수 원칙
- 파일이 과도하게 길어지면(가이드 기준 18KB 초과) **2~3개로 분할**한다

---

## 7) 코드/품질/보안 공통 규칙
- 비밀정보(키/토큰/서비스 계정 JSON)는 **코드/문서/커밋에 포함 금지**
- 새 의존성 추가는 “왜 필요한지/대체안/리스크”를 결재 문맥으로 설명
- 테스트/검증은 가능한 한 재현 가능하게 기록(명령어/클릭 플로우/로그)

(보안 사고 의심 시 표준 대응)
- 1) 노출 범위 파악(어디에 포함됐는지)
- 2) 키/토큰 **즉시 폐기 및 재발급(rotate)**
- 3) 재발 방지 규칙 문서에 추가(append)

## 8) HR(감리) 체크리스트 (통과/실패 판정 기준)
아래 항목 중 하나라도 누락되면 **“미통과(재보고 요청)”**.

### 8-1) 착수 전
- [ ] 결재 필요 여부를 확인했고, 필요 시 보고서로 올렸다
- [ ] 작업 범위/가정/리스크가 1~3줄로 명시됐다

### 8-2) 수행 중
- [ ] 오류가 나면 원인→옵션→추천 순서로 보고했다(무단 조치 없음)
- [ ] 불필요하게 범위를 키우지 않았다(최소 변경)

### 8-3) 완료 후
- [ ] 변경 요약(핵심 3줄)
- [ ] 테스트/검증 결과
- [ ] 영향 범위 명시
- [ ] 산출물 경로/링크(보고서/회의록/커밋 등)
- [ ] 문서 동기화가 필요한 경우 append로 반영
