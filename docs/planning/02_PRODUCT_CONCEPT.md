## 제품 컨셉(초안)

## 1) 목적
- **특정 컨텐츠(예: 음악/영상/텍스트/이미지/링크)**를 매개로 사용자가 자신의 생각과 감정을 기록하고, 시간이 지나도 다시 꺼내 볼 수 있는 아카이브를 만든다.

## 1-1) 중심 컨텐츠 범위(1차, 결재 반영)
- **YouTube + SoundCloud**로 한정
- **확장 여지**: 플랫폼 추가는 결재 후 범위를 확장(구조/정책은 확장 가능하게 설계)

## 2) 핵심 객체(데이터/콘텐츠 단위)
- **Entry(기록)**: 제목, 본문(생각/감정), 작성자(선택), 작성일
- **Content(중심 컨텐츠)**: URL/메타(타이틀/썸네일/플랫폼/아티스트 등) — (현재는 기획 정의만)
- **Mood/Tags(정서/주제 태그)**: 검색/회고/큐레이션을 위한 메타

## 3) 사용자 가치 흐름(가치가 생기는 순간)
- 컨텐츠를 본다/듣는다 → 감정이 생긴다 → 기록한다 → 쌓인다 → 다시 본다(회고) → 연결된다(연속 기록/테마)

## 4) 톤 & 무드(현재 UI와 합치)
- System Node / Archive / Pixel / Sentimental
- UI 문구는 "기록"을 "아카이브 생성"으로 치환하여 세계관 일관성을 유지

## 5) 범위(현재 단계)
- **Must**: 기록(작성/목록/상세) 경험이 매끄럽고, 컨텐츠 중심 기록이라는 목적을 설명/유도한다.
- **Later**: 컨텐츠 메타 자동 수집, 감정 태깅 보조, 회고/검색/큐레이션
