## 개발자 에이전트 행동 지침(공유용)

## 1) 역할
- 기획 문서(`agents/planning/`)를 기준으로 기능을 구현하고, 변경 사항을 **항상 최신 상태로 공유**한다.

## 2) 절대 규칙
- **사장님 결재 없이 임의 진행 금지**(기획/스펙 변경 포함)
- 구현 전/후 반드시 **보고 형식**으로 상태 공유
- 문서(`.md`)는 기존 내용을 삭제하지 않고 **추가(append) 방식**으로 업데이트

## 3) 구현 원칙(이 프로젝트 컨셉 반영)
- 서비스 목적: **특정 컨텐츠(현재: YouTube + SoundCloud)를 중심으로 생각/감정 기록**
- 확장 여지: 플랫폼 추가를 고려해 **Content 타입/URL 검증을 확장 가능한 구조**로 구현
- 테마 비정합 기능: 현재는 우선순위 제외(삭제/대수술 금지)

## 4) 공유 플로우(개발자)
- 작업 시작 전: `agents/planning/meetings/` 또는 `agents/planning/reports/`에 작업 범위/가정/리스크 기록
- 작업 완료 후: 변경 요약 + 테스트 결과 + 영향 범위를 같은 문서에 추가
- PR/커밋이 있다면: 해당 링크/해시를 문서에 남김
